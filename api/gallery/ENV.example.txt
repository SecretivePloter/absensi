MODE AUTH GOOGLE DRIVE — pilih SALAH SATU:

[A] OAUTH (DIUTAMAKAN, untuk simpan ke Drive Gmail personal spt mugenworklabs@gmail.com)
    File dimiliki akun user & pakai kuota Drive-nya (tak kena error kuota
    service account). Ambil nilainya dengan menjalankan:
        node scripts/get-drive-token.mjs
    (lihat instruksi di dalam file itu). Lalu isi:

    GOOGLE_OAUTH_CLIENT_ID
      Client ID OAuth (tipe Desktop app) dari Cloud Console.
      contoh: 1234-abcd.apps.googleusercontent.com

    GOOGLE_OAUTH_CLIENT_SECRET
      Client secret dari OAuth Client tersebut.

    GOOGLE_OAUTH_REFRESH_TOKEN
      Hasil cetakan script get-drive-token.mjs (sekali buat, long-lived
      selama consent screen "In production").

    GDRIVE_FOLDER_ID
      ID folder yang dibuat otomatis oleh script get-drive-token.mjs.

[B] SERVICE ACCOUNT (fallback, HANYA untuk Shared Drive / Workspace)
    GDRIVE_CLIENT_EMAIL   email service account (...@...iam.gserviceaccount.com)
    GDRIVE_PRIVATE_KEY    field "private_key" dari JSON (boleh \n literal)
    GDRIVE_FOLDER_ID      id folder di Shared Drive

  Catatan: bila GOOGLE_OAUTH_REFRESH_TOKEN diisi, mode [A] otomatis dipakai.
